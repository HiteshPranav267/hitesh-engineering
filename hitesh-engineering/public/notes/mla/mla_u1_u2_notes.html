1.1 Introduction to Machine Learning

Definition
Machine Learning is the science of getting computers to learn and improve from experience without being explicitly programmed. It provides a new capability of computers to mimic human decision-making.

Why Machine Learning?

  • Abundant data — massive datasets available from the web, sensors, IoT
  • Mature models & algorithms — well-studied theoretical foundations
  • Fast & cheap computing — GPUs, cloud platforms enable large-scale training
  • Hard-to-program tasks — face recognition, speech recognition, medical image analysis cannot easily be written as rule-based programs

Instead of hand-writing a program, we collect lots of examples that specify the correct output for a given input. A machine learning algorithm takes these examples and produces a program (model) that generalizes to new unseen cases.

Classic Example — Handwritten Digit Recognition

Input: 28×28 pixel image = 784-dimensional vector $\mathbf{x}$

Training set: $\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_N$ with labels $t \in \{0, 1, \ldots, 9\}$

Goal: learn a function that accepts any new image $\mathbf{x}$ and predicts the digit (0-9). The model must generalize to unseen handwritten inputs.

Key Applications of ML

Domain Application
Pattern Recognition Facial identity/expression, handwritten/spoken words, medical images
Anomaly Detection Unusual credit card transactions, sensor readings in nuclear plants
Prediction Stock prices, weather, currency exchange rates
Web Applications Spam filtering, fraud detection, recommendation systems
Generation Image generation, motion sequences, text generation
Information Retrieval Finding documents/images with similar content

1.2 Types of Machine Learning

Supervised Learning

Learn to predict output given an input vector. Training data consists of input-output pairs $(x, y)$.

  • Classification: output is a discrete class label
  • Regression: output is a continuous numeric value
Unsupervised Learning

No labels given. Create an internal representation of the input (e.g., clusters, features).

  • Clustering: group similar data points (K-Means)
  • Dimensionality Reduction: PCA, Autoencoders
Reinforcement Learning

Learn actions to maximize cumulative reward through trial and error. Payoff is often delayed.

Classification vs Regression

Classification: disease diagnosis (K classes), spam/not-spam (2 classes)

Regression: house price, sales forecasting, age estimation

Left: Supervised learning with labeled classes and a decision boundary. Right: Unsupervised learning discovers clusters without labels.

1.3 Training, Validation & Test Sets

In supervised ML, we split data into three distinct portions to ensure the model learns, is tuned correctly, and is evaluated fairly.

Training Set (60–70%)

Used to teach the model. The model fits its parameters (e.g., weights in regression, tree splits) by minimizing errors on this set using an optimization algorithm.

Validation Set (15–20%)

Used to tune hyperparameters and select the best model. Adjust learning rate, number of layers, $k$ in k-NN, etc. Prevents overfitting to training data. Supports early stopping decisions.

Test Set (10–15%)

Used only once, at the end for final unbiased evaluation. No learning or tuning is done on test data. Gives a realistic measure of how the model performs on truly unseen data.

Important

Never use the same dataset for both training and validation/testing. Using the same data leads to biased, overly optimistic results because the model memorizes the answers rather than learning generalizable patterns.

Over the validation set, a single run is not enough because: (1) Training and validation sets may be small and contain noise/outliers, (2) The learning method may depend on random factors (e.g., initial weights in MLP). Run many times and average for robustness.

Example — Student Pass/Fail Prediction

Task: Predict whether a student will Pass (1) or Fail (0) based on hours studied.

Training: model learns that low study hours → Fail, higher hours → Pass. Decision boundary learned ≈ 4.5 hours.

Validation: test different thresholds (4 or 5 hours), choose the one with better accuracy.

Test: evaluate on held-out data. Test accuracy = 100% confirms the model generalizes well.

from sklearn.datasets import make_blobs
from sklearn.model_selection import train_test_split

X, y = make_blobs(n_samples=1000)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.33)
print(X_train.shape, X_test.shape)   # (670, 2) (330, 2)

1.4 Learning a Class from Examples

Consider learning to classify family cars. Each car is represented by two features: Price ($x_1$) and Engine Power ($x_2$).

Problem Setup
  • Each car is a vector $\mathbf{x} = \begin{pmatrix} x_1 \\ x_2 \end{pmatrix}$ where $x_1 = \text{Price}$, $x_2 = \text{Engine Power}$
  • Label: $r = 1$ if $\mathbf{x}$ is a positive example (family car), $r = 0$ if negative
  • Training set: $X = \{(\mathbf{x}^t, r^t)\}_{t=1}^{N}$
  • The actual class $C$ defines a region in feature space where the car is a family car

Hypothesis Class

We assume the hypothesis class $H$ consists of axis-aligned rectangles defined by four parameters:

  • $p_1$ = lower limit of price, $p_2$ = upper limit of price
  • $e_1$ = lower limit of engine power, $e_2$ = upper limit of engine power
Hypothesis
$$h(\mathbf{x}) = \begin{cases} 1 & \text{if } p_1 \leq \text{Price} \leq p_2 \text{ AND } e_1 \leq \text{Engine Power} \leq e_2 \\ 0 & \text{otherwise} \end{cases}$$
Family car classification: S is the tightest rectangle around positive examples, G is the widest rectangle excluding negatives. The version space lies between S and G.

Empirical Error

Since the true class $C(\mathbf{x})$ is unknown, we evaluate the empirical error — the proportion of training instances where the hypothesis disagrees with the label:

Empirical Error
$$E(h|X) = \sum_{t=1}^{N} \mathbf{1}(h(\mathbf{x}^t) \neq r^t)$$

where $\mathbf{1}(\cdot)$ is the indicator function: returns 1 if condition is true, 0 otherwise. We seek $h$ such that $E = 0$.

1.5 Version Space, Doubt & Margin

Most Specific (S) & Most General (G) Hypotheses

S = tightest rectangle that includes all positive examples
G = widest rectangle that excludes all negative examples
Any $h \in H$ between S and G has zero error on training data — it is consistent with the training set. This set of consistent hypotheses forms the Version Space.

Example

S: $8 \leq \text{price} \leq 10$ AND $4 \leq \text{engine power} \leq 6$

G: $4 \leq \text{price} \leq 14$ AND $2 \leq \text{engine power} \leq 8$

C (actual): $4 \leq \text{price} \leq 13$ AND $2.5 \leq \text{engine power} \leq 7.5$

False Positive & False Negative

If $h$ and $C$ are different:

  • False Negative (FN): $C = 1$ but $h = 0$ (point is in C but outside h). Example: Price=12.75 lakhs, Engine Power=4
  • False Positive (FP): $C = 0$ but $h = 1$ (point is outside C but inside h). Example: Price=10 lakhs, Engine Power=7.9

Doubt & Rejection

When an instance falls between S and G, we have a case of doubt — there isn't enough data to classify it. The system rejects such instances and defers the decision to a human expert.

Margin

We choose the hypothesis with the largest margin for best separation. Margin instances are those that define the boundary; other instances can be removed without affecting $h$.

Candidate Elimination Algorithm

The Candidate Elimination algorithm incrementally updates the S-Set and G-Set as it sees training instances one by one. If the training set is large enough, there is a unique S and G.

1.6 Vapnik-Chervonenkis (VC) Dimension

Definition

The VC Dimension of a hypothesis class $H$ is the maximum number of data points that can be shattered by $H$. We write $\text{VC}(H)$.

Shattering: A set of $n$ points is shattered by $H$ if for all $2^n$ possible labelings (positive/negative), there exists some $h \in H$ that correctly separates them.

Why VC Dimension Matters
  • Measures model complexity/capacity
  • Helps understand overfitting vs. generalization
  • Provides theoretical guarantees on learning
  • Higher VC dimension ⇒ more training data required to generalize

VC Dimension Examples

Hypothesis Class VC Dimension Reasoning
Threshold on a line: $h(x) = 1 \text{ if } x \geq c$ 1 Can shatter 1 point; fails for 2 points in some configurations
Linear classifier in 2D (line) 3 3 non-collinear points: all $2^3 = 8$ labelings achievable. 4 points: XOR labeling impossible
Linear classifier in $d$ dimensions $d + 1$ A hyperplane in $d$-D can shatter $d+1$ points
Axis-aligned rectangle in 2D 4 Can shatter 4 corner-like points with rectangles
Left: A linear classifier in 2D can shatter 3 non-collinear points. Right: 4 points in XOR configuration cannot be separated by any line, so VC dim = 3 (not 4).
Key Insight

Low VC dimension → simple model → may underfit
High VC dimension → complex model → risk of overfitting
Generalization error depends on: training error, VC dimension, and number of training samples.

1.7 Probably Approximately Correct (PAC) Learning

Definition

PAC Learning is a theoretical framework that defines when a learning algorithm can be considered successful. A concept class is PAC-learnable if there exists an algorithm producing a hypothesis that is approximately correct (error $\leq \varepsilon$) with high probability ($\geq 1 - \delta$) using a finite number of samples.

Formal Statement

We want to find the number of examples $N$ such that with probability $(1 - \delta)$, the hypothesis $h$ has error at most $\varepsilon$:

PAC Condition
$$P(C \Delta h \leq \varepsilon) \geq 1 - \delta$$

where $C \Delta h$ = region of difference between actual class C and hypothesis h.

Derivation for Rectangle Hypothesis

Using $h = S$ (tightest rectangle), the error region between C and S is the sum of four rectangular strips.

  1. Probability each positive example falls in one strip = $\varepsilon/4$
  2. Probability each draw misses one strip = $(1 - \varepsilon/4)$
  3. Probability $N$ independent draws all miss one strip = $(1 - \varepsilon/4)^N$
  4. Probability misses at least one of four strips $\leq 4(1 - \varepsilon/4)^N \leq 4 e^{-N\varepsilon/4}$

Setting $4 e^{-N\varepsilon/4} \leq \delta$ and solving:

Sample Complexity
$$N \geq \frac{4}{\varepsilon} \ln \frac{4}{\delta}$$
Numerical Example

Given: $\varepsilon = 0.05$ (5% error allowed), $\delta = 0.01$ (99% confidence)

Calculate:

$$N \geq \frac{4}{0.05} \ln \frac{4}{0.01} = 80 \times \ln(400) = 80 \times 5.991 = 479.3$$

Result: We need at least $N = 480$ training examples. With 99% probability, the learned model's error will be $\leq$ 5% on unseen data.

Key Properties

• As $\delta$ decreases (higher confidence), $N$ increases — more data needed for greater certainty

• As $\varepsilon$ decreases (tighter accuracy), $N$ increases — more data needed for better precision

• With VC dimension $d$: $m = O\left(\frac{1}{\varepsilon}\left(d \log \frac{1}{\varepsilon} + \log \frac{1}{\delta}\right)\right)$

1.8 Noise & Occam's Razor

Definition

Noise is an unwanted anomaly in the data. Due to noise, the class may be difficult to learn and zero error may be impossible with a simple hypothesis class.

Sources of Noise

  1. Input imprecision: Recording errors in input attributes shift data points in input space
  2. Teacher's noise: Errors in labeling — positive examples mislabeled as negative (and vice versa)
  3. Hidden variables: Unobserved attributes that affect the label are modeled as random noise

Dealing with Noise

When there is noise, there isn't a clean boundary between classes. A complex hypothesis with many parameters could perfectly separate noisy data, but this is undesirable.

Occam's Razor

Given comparable empirical error, a simple model generalizes better than a complex one. A rectangle (4 parameters) is preferred over an arbitrary closed form (many control points) because:

  1. Simple to use — easy to check if a point is inside/outside
  2. Simple to train — fewer parameters, less variance
  3. Simple to explain — defines intervals on attributes
  4. Better generalization in the presence of noise

1.9 Learning Multiple Classes

With $K$ classes $C_1, C_2, \ldots, C_K$ (e.g., Family Car, Luxury Car, Sports Car), the training set is $X = \{(\mathbf{x}^t, \mathbf{r}^t)\}_{t=1}^{N}$ where $\mathbf{r}$ has $K$ dimensions:

One-Hot Encoding
$$r_i^t = \begin{cases} 1 & \text{if } \mathbf{x}^t \in C_i \\ 0 & \text{if } \mathbf{x}^t \in C_j, \, j \neq i \end{cases}$$

We learn $K$ hypotheses $h_1, h_2, \ldots, h_K$ where each $h_i$ separates class $C_i$ from all other classes. The total empirical error:

Total Error (K classes)
$$E(\{h_i\}_{i=1}^{K} | X) = \sum_{t=1}^{N} \sum_{i=1}^{K} \mathbf{1}(h_i(\mathbf{x}^t) \neq r_i^t)$$

For a given $\mathbf{x}$, ideally only one $h_i(\mathbf{x}) = 1$ and we choose that class. If two or more $h_i = 1$, we have ambiguity. If no $h_i = 1$, we have doubt and the classifier rejects.

1.10 Regression (Introduction)

Definition

Regression predicts a continuous numeric output as a function of input variables. Unlike classification (Boolean output), regression output can be any real number.

Example — Sales Prediction
Year Advertisement ($) Sales ($)
2021 90 1000
2022 100 1050
2023 150 1800
2024 100 1200
2025 130 1380
2026 200 ?

Univariate linear model: Sales $= w_0 + w_1 \times$ Advertisement

Quadratic model: Sales $= w_0 + w_1 \times$ Ad $+ w_2 \times$ Ad$^2$

Training = finding the parameter values ($w_0, w_1, \ldots$) that best fit the data.

Univariate vs. Multivariate

If sales depends on one variable (advertisement) → univariate. If sales depends on advertisement, fuel price, and interest rate → multivariate:

$$\text{Sales} = w_0 + w_1 \cdot \text{Ad} + w_2 \cdot \text{Interest Rate} + w_3 \cdot \text{Fuel Price}$$

Key Point

As we choose higher-order models or work with more features, we need larger datasets to reliably determine the parameters. The model order and the dataset size are critical factors in ML.

1.11 Model Selection & Generalization

Definition

Generalization is a model's ability to perform accurately on new, unseen data. Good generalization = low error on both training and new data. Poor generalization = overfitting (too complex) or underfitting (too simple).

Ill-Posed Problem & Inductive Bias

For a Boolean function with $d$ inputs: there are $2^d$ possible input combinations and $2^{2^d}$ possible functions. Each training example eliminates half the hypotheses. With $N < 2^d$ examples, $2^{2^d - N}$ functions remain — this is ill-posed (data alone isn't sufficient for a unique solution).

Inductive Bias

The set of assumptions we make to have learning possible is called inductive bias. Examples:

  • Assuming the class boundary is a rectangle → one inductive bias
  • Choosing the rectangle with largest margin → another bias
  • Assuming a linear function in regression → linear bias

Model selection = choosing the right inductive bias (choosing between possible $H$).

Underfitting vs. Overfitting

Underfitting (High Bias)

$H$ is less complex than the true function. E.g., fitting a line to cubic data. Training error is high. The model fails to capture essential patterns.

Overfitting (High Variance)

$H$ is more complex than needed. E.g., fitting a degree-6 polynomial to noisy cubic data. Training error is near zero, but test error is high. The model memorizes noise.

Three polynomial fits to the same data: underfitting (linear), good fit (appropriate degree), overfitting (too high degree).

Triple Trade-Off

The three factors in tension:

  1. Complexity of hypothesis class (capacity of $H$)
  2. Amount of training data ($N$)
  3. Generalization error on new examples

1.12 Dimensions of a Supervised ML Algorithm

Given training set $X = \{(\mathbf{x}^t, r^t)\}_{t=1}^{N}$, to build a useful approximation to $r^t$ using model $g(\mathbf{x}^t | \theta)$, we need three decisions:

1. Model $g(\mathbf{x}|\theta)$

$g(\cdot)$ defines the hypothesis class $H$. A particular value of $\theta$ instantiates one hypothesis $h \in H$.

  • Rectangle classifier: four coordinates are $\theta$
  • Linear regression: slope $m$ and intercept $c$ are $\theta$

The model (inductive bias) is fixed by the designer. The hypothesis is tuned by the learning algorithm using training data sampled from $p(\mathbf{x}, r)$.

2. Loss Function $L(\cdot)$

Computes the difference between desired output $r^t$ and approximation $g(\mathbf{x}^t|\theta)$:

$$E(\theta|X) = \sum_{t=1}^{N} L(r^t, g(\mathbf{x}^t|\theta))$$

  • Classification: $L$ checks equality (0/1 loss)
  • Regression: $L$ uses squared difference: $L = (r^t - g(\mathbf{x}^t|\theta))^2$
3. Optimization Procedure

Find $\theta^* = \arg\min_\theta E(\theta|X)$

In polynomial regression we can solve analytically. In complex models, we use iterative methods (e.g., gradient descent). Key question: does it have a single minimum (global) or multiple minima (local)?

Requirements for Success
  1. $H$ must be large enough to include the unknown function (sufficient capacity)
  2. Enough training data to pinpoint the correct hypothesis
  3. A good optimization method to find the correct hypothesis

1.13 Confusion Matrix & Performance Metrics

Confusion Matrix

Predicted Positive Predicted Negative
Actual Positive True Positive (TP) False Negative (FN) — Type 2 error
Actual Negative False Positive (FP) — Type 1 error True Negative (TN)
Numerical Example

Dataset: 1000 data points. Confusion matrix:

Predicted + Predicted −
Actual + TP = 560 FN = 50
Actual − FP = 60 TN = 330

Accuracy = $\frac{TP + TN}{TP + TN + FP + FN} = \frac{560 + 330}{1000} = \frac{890}{1000} = 0.89 = 89\%$

Precision (PPV) = $\frac{TP}{TP + FP} = \frac{560}{560 + 60} = \frac{560}{620} = 0.903 = 90.3\%$

Recall (Sensitivity/TPR) = $\frac{TP}{TP + FN} = \frac{560}{560 + 50} = \frac{560}{610} = 0.918 = 91.8\%$

Specificity (TNR) = $\frac{TN}{TN + FP} = \frac{330}{330 + 60} = \frac{330}{390} = 0.846 = 84.6\%$

F1-Score = $\frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}} = \frac{2 \times 0.903 \times 0.918}{0.903 + 0.918} = \frac{1.658}{1.821} = 0.910 = 91.0\%$

FPR = $1 - \text{Specificity} = 1 - 0.846 = 0.154 = 15.4\%$

FNR = $1 - \text{Recall} = 1 - 0.918 = 0.082 = 8.2\%$

Summary of All Metrics

Metric Formula Interpretation
Accuracy $\frac{TP + TN}{TP+TN+FP+FN}$ Overall correctness
Precision (PPV) $\frac{TP}{TP + FP}$ Of predicted positives, how many correct?
Recall (Sensitivity/TPR) $\frac{TP}{TP + FN}$ Of actual positives, how many detected?
Specificity (TNR) $\frac{TN}{TN + FP}$ Of actual negatives, how many detected?
F1-Score $\frac{2 \cdot P \cdot R}{P + R}$ Harmonic mean of precision & recall
False Positive Rate (FPR) $\frac{FP}{FP + TN} = 1 - \text{Specificity}$ Proportion of negatives wrongly classified
Jaccard Similarity Index $\frac{TP}{TP + FN + FP}$ Intersection over union of predicted & actual

1.14 ROC Curve & AUC

By varying the decision threshold $\theta$, we get different (TPR, FPR) pairs. Plotting these creates the Receiver Operating Characteristic (ROC) curve.

ROC Curve: The closer to the upper-left corner (0, 1), the better. AUC (Area Under Curve) summarizes performance as a single number (perfect = 1.0, random = 0.5).
Key ROC Interpretations
  • A classifier with TPR = 1 and FPR = 0 is perfect → upper-left corner
  • One classifier is better if its ROC curve is above the other
  • If two curves intersect, each is better under different loss conditions
  • AUC reduces the curve to a single number for easy comparison

1.15 Precision & Recall (Information Retrieval View)

In information retrieval: query a database and retrieve records.

Precision

$$\text{Precision} = \frac{\text{Retrieved} \cap \text{Relevant}}{\text{Retrieved}} = \frac{TP}{TP + FP}$$

If precision = 1, all retrieved records are relevant, but some relevant records may be missed.

Recall

$$\text{Recall} = \frac{\text{Retrieved} \cap \text{Relevant}}{\text{Relevant}} = \frac{TP}{TP + FN}$$

If recall = 1, all relevant records are retrieved, but irrelevant records may also be retrieved.

Sensitivity = TPR = Recall. Specificity = how well we detect negatives = $TN / (TN + FP)$.

Unit 2: Supervised Learning — Parametric Methods

Parametric Classification, Regression, Bias-Variance, Model Selection, Multivariate Methods, Gradient Descent, Logistic Discrimination

2.1 Parametric vs. Non-Parametric Methods

Parametric Methods

Assume a fixed form for the model and summarize data using a finite set of parameters. Once learned, the model doesn't need the original dataset.

  • Assume known distribution (e.g., Gaussian)
  • Parameters: mean $\mu$, standard deviation $\sigma$
  • Central Limit Theorem applicable
  • Examples: Linear regression, Logistic regression, Gaussian Naive Bayes
Non-Parametric Methods

Make no assumptions about the form of the distribution. No fixed set of parameters — also called distribution-free methods.

  • No distributional assumptions needed
  • Lower complexity, easy to apply
  • Examples: k-NN, Kernel Density Estimation, decision trees
  • Spearman correlation, Sign test, U-test

2.2 Regression (Detailed Treatment)

Given i.i.d. samples, output variables are functions of the input (independent variables):

Regression Model
$$r^t = f(\mathbf{x}^t) + \epsilon$$

$f(\mathbf{x})$ = unknown function, $\epsilon \sim \mathcal{N}(0, \sigma^2)$ = Gaussian noise with zero mean and constant variance.

We approximate $f$ by our estimator $g(\mathbf{x}|\theta)$. The empirical error on training set $X$:

Mean Squared Error
$$E(g|X) = \frac{1}{N} \sum_{t=1}^{N} [r^t - g(\mathbf{x}^t)]^2$$

Maximum Likelihood & Least Squares

Given Gaussian noise, the joint probability density is $p(\mathbf{x}, r)$. The log-likelihood is:

$$\ell = \sum_{t=1}^{N} \log p(r^t | \mathbf{x}^t, \theta) + \sum_{t=1}^{N} \log p(\mathbf{x}^t)$$

The second term is independent of $\theta$. Since the likelihood contains exponents, instead of maximizing $\ell$, we define $E = -\log \ell$ and minimize it. This gives us the Least Squares Estimates.

Linear Regression

The simplest model: $g(\mathbf{x}) = w_1 x + w_0$

Normal Equations (Linear)

Minimize $E = \sum_t (r^t - w_1 x^t - w_0)^2$ by taking partial derivatives and setting to zero:

$$w_1 = \frac{N \sum_t x^t r^t - \sum_t x^t \sum_t r^t}{N \sum_t (x^t)^2 - (\sum_t x^t)^2}$$ $$w_0 = \bar{r} - w_1 \bar{x}$$

where $\bar{x}$ = mean of inputs, $\bar{r}$ = mean of outputs.

Numerical — Simple Linear Regression

Data: $(1, 2), (2, 4), (3, 5), (4, 4), (5, 5)$

$N = 5$, $\sum x = 15$, $\sum r = 20$, $\sum xr = 67$, $\sum x^2 = 55$

$$w_1 = \frac{5(67) - 15(20)}{5(55) - 15^2} = \frac{335 - 300}{275 - 225} = \frac{35}{50} = 0.7$$

$$w_0 = \frac{20}{5} - 0.7 \times \frac{15}{5} = 4 - 2.1 = 1.9$$

Model: $g(x) = 0.7x + 1.9$

Predict $x = 6$: $g(6) = 0.7(6) + 1.9 = 6.1$

2.3 Polynomial Regression

If the linear model is too simple, we use a polynomial of order $k$:

Polynomial Model
$$g(x) = w_k x^k + w_{k-1} x^{k-1} + \cdots + w_1 x + w_0$$

In vector form, define:

$$\mathbf{D} = \begin{pmatrix} 1 & x^1 & (x^1)^2 & \cdots & (x^1)^k \\ 1 & x^2 & (x^2)^2 & \cdots & (x^2)^k \\ \vdots & & & & \vdots \\ 1 & x^N & (x^N)^2 & \cdots & (x^N)^k \end{pmatrix}, \quad \mathbf{w} = \begin{pmatrix} w_0 \\ w_1 \\ \vdots \\ w_k \end{pmatrix}, \quad \mathbf{r} = \begin{pmatrix} r^1 \\ r^2 \\ \vdots \\ r^N \end{pmatrix}$$

Closed-Form Solution
$$\mathbf{w} = (\mathbf{D}^T \mathbf{D})^{-1} \mathbf{D}^T \mathbf{r}$$

Goodness of Fit

Relative Squared Error & R²
$$E_{RSE} = \frac{\sum_t (r^t - g^t)^2}{\sum_t (r^t - \bar{r})^2}$$ $$R^2 = 1 - E_{RSE}$$

If $E_{RSE} = 1$: prediction is as good as predicting the average. If $E_{RSE} = 0$: perfect fit. For regression to be useful, $R^2$ should be close to 1.

2.4 Bias / Variance Dilemma

Consider samples drawn from joint density $X = \{(\mathbf{x}^t, r^t)\}$. The expected squared error at $\mathbf{x}$ can be decomposed:

Bias-Variance Decomposition
$$\text{Expected Error} = \underbrace{\sigma^2}_{\text{Noise (irreducible)}} + \underbrace{(\mathbb{E}[g(\mathbf{x})] - f(\mathbf{x}))^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}[(g(\mathbf{x}) - \mathbb{E}[g(\mathbf{x})])^2]}_{\text{Variance}}$$
Three Components
  1. Noise ($\sigma^2$): irreducible error from data noise; independent of the estimator
  2. Bias: measures how much $g(\mathbf{x})$ deviates from the true function $f(\mathbf{x})$ on average across datasets. High bias = wrong assumptions = underfitting
  3. Variance: measures how much $g(\mathbf{x})$ fluctuates around its expected value across different training sets. High variance = too sensitive to data = overfitting
Bias-Variance Trade-off: As model complexity increases, bias decreases but variance increases. The optimal model minimizes total error.
The Dilemma

To decrease bias → make model more flexible → risk of high variance

To keep variance low → use a simpler model → risk of high bias

The optimal model has the best trade-off. If variance is large but bias is low, averaging many high-variance models can reduce error (ensemble methods).

Example — Polynomial Fit Comparison

True function: $f(x) = 2\sin(1.5x)$ with noise $\mathcal{N}(0, 1)$. Five samples of 20 instances each:

Polynomial Order Bias Variance Assessment
1 (linear) High Low Underfitting — can't capture the sine curve
3 (cubic) Low Moderate Good fit — close to the true function
5 (quintic) Very Low Very High Overfitting — small data changes cause large changes in the fit

2.5 Model Selection Procedures

1. Cross-Validation

Divide data into training set and validation set. Train models of different complexities on training data, evaluate on validation set. As complexity increases, training error monotonically decreases, but validation error follows a U-shape. The elbow corresponds to the optimal complexity.

Cross-validation: training error always decreases, but validation error has a minimum at the optimal complexity level.

2. Regularization

Add a penalty term for complex models:

Augmented Error
$$E_{aug} = E_{data} + \lambda \|\mathbf{w}\|^2$$

The second term penalizes large parameter values (complex models). $\lambda$ controls the trade-off. If $\lambda$ too large → too simple (bias risk). $\lambda$ is optimized via cross-validation.

3. Akaike's Information Criterion (AIC) & Bayesian Information Criterion (BIC)

Estimate the optimism (gap between training and test error) and add it to training error. No validation needed.

  • Optimism increases linearly with $d$ (number of inputs)
  • Decreases with $N$ (training set size)
  • Increases with $\sigma^2$ (noise variance)
  • For nonlinear models, replace $d$ with effective number of parameters

4. Structural Risk Minimization (SRM)

Use a set of models ordered by increasing complexity (e.g., polynomials of increasing order, decreasing $\lambda$). Model selection finds the model simplest in order and best in empirical error.

5. Minimum Description Length (MDL)

Based on Kolmogorov complexity: the shortest description of the data. If data is simple (e.g., sequence of 0s), it has short description. Among all fitting models, choose the simplest one that gives the shortest description. Trade-off between model simplicity and data description.

6. Bayesian Model Selection

$$p(\text{model}|\text{data}) \propto p(\text{data}|\text{model}) \cdot p(\text{model})$$

Taking logarithm: log likelihood (training error) + log prior (penalty term). With prior $p(\mathbf{w}) \sim \mathcal{N}(0, 1/\lambda)$, minimizing corresponds to regularization. When giving higher probability to simpler models (Occam's razor), Bayesian approach, regularization, SRM, and MDL are equivalent.

Key Comparison

Cross-validation makes no prior assumptions and is best with large validation data. The other methods are more useful when the data sample is small.

2.6 Parametric Classification

We use Bayes' rule to classify. The posterior probability of class $C_i$ given input $\mathbf{x}$:

Bayes' Rule for Classification
$$P(C_i|\mathbf{x}) = \frac{p(\mathbf{x}|C_i)P(C_i)}{p(\mathbf{x})}$$

The discriminant function is: $g_i(\mathbf{x}) = \log p(\mathbf{x}|C_i) + \log P(C_i)$

We assign $\mathbf{x}$ to class $C_i$ if $g_i(\mathbf{x}) = \max_j g_j(\mathbf{x})$.

Assuming Gaussian Class-Conditional Densities

If $p(\mathbf{x}|C_i) \sim \mathcal{N}(\mu_i, \sigma_i^2)$, then:

$$g_i(x) = -\frac{(x - \mu_i)^2}{2\sigma_i^2} - \frac{1}{2}\log \sigma_i^2 + \log P(C_i)$$
MLE for Parameters

$$\hat{\mu}_i = \frac{\sum_{t: \mathbf{x}^t \in C_i} x^t}{|C_i|}, \quad \hat{\sigma}_i^2 = \frac{\sum_{t: \mathbf{x}^t \in C_i} (x^t - \hat{\mu}_i)^2}{|C_i|}, \quad \hat{P}(C_i) = \frac{|C_i|}{N}$$

Case 1: Equal Variances ($\sigma_1^2 = \sigma_2^2 = \sigma^2$)

The discriminant simplifies. The constant term $-\frac{1}{2\sigma^2}$ and $\log \sigma^2$ cancel. If priors are equal, the sample $x$ is assigned to the class with the nearest mean. With two classes, the threshold is at $x_0 = \frac{\mu_1 + \mu_2}{2}$. One intersection point between the posteriors.

Case 2: Unequal Variances

The discriminant function becomes quadratic in $x$. The posteriors intersect at two points, creating two thresholds. Between them, one class dominates; outside, the other class dominates.

Left: Equal variance Gaussians intersect at one point. Right: Unequal variance creates two intersection points (two thresholds).
Numerical — Parametric Classification (Equal Variance)

Given: Two classes with equal priors $P(C_1) = P(C_2) = 0.5$ and equal variances $\sigma^2 = 4$.

$\mu_1 = 3$, $\mu_2 = 7$

Threshold: $x_0 = \frac{\mu_1 + \mu_2}{2} = \frac{3 + 7}{2} = 5$

Decision: If $x < 5 \Rightarrow C_1$, if $x> 5 \Rightarrow C_2$

For $x = 4$: classify as $C_1$ (since $4 < 5$). ✓

For $x = 6$: classify as $C_2$ (since $6 > 5$). ✓

2.7 Multivariate Data & Parameter Estimation

With multiple inputs, data is represented as an $N \times d$ matrix $\mathbf{X}$ where $d$ columns = features/attributes and $N$ rows = observations/instances.

Mean Vector & Covariance Matrix
$$\boldsymbol{\mu} = \begin{pmatrix} \mu_1 \\ \mu_2 \\ \vdots \\ \mu_d \end{pmatrix}, \quad \boldsymbol{\Sigma} = \begin{pmatrix} \sigma_{11} & \sigma_{12} & \cdots & \sigma_{1d} \\ \sigma_{21} & \sigma_{22} & \cdots & \sigma_{2d} \\ \vdots & & \ddots & \vdots \\ \sigma_{d1} & \sigma_{d2} & \cdots & \sigma_{dd} \end{pmatrix}$$

Diagonal elements = variances $\sigma_i^2$. Off-diagonal = covariances $\sigma_{ij} = \sigma_{ji}$. Matrix is symmetric. With $d$ variables: $d$ variances and $d(d-1)/2$ covariances.

The correlation between $X_i$ and $X_j$ is: $\rho_{ij} = \frac{\sigma_{ij}}{\sigma_i \sigma_j}$, normalized between $-1$ and $+1$.

Sample Estimates (MLE)

$$\mathbf{m} = \frac{1}{N}\sum_{t=1}^{N} \mathbf{x}^t, \quad \mathbf{S} = \frac{1}{N}\sum_{t=1}^{N} (\mathbf{x}^t - \mathbf{m})(\mathbf{x}^t - \mathbf{m})^T$$

In matrix notation: $\mathbf{S} = \frac{1}{N}\mathbf{X}^T\mathbf{X} - \mathbf{m}\mathbf{m}^T$

Estimation of Missing Values (Imputation)

Imputation Methods
  • Mean imputation: For numeric variables, substitute the mean of available data for that variable
  • Mode imputation: For discrete variables, fill with the most likely (most frequent) value
  • Regression imputation: Predict the missing value from other variables whose values are known. If many variables are missing, start with means as initial estimates and iterate until values stabilize. If variables aren't highly correlated, regression imputation ≈ mean imputation.

2.8 Multivariate Normal Distribution

Multivariate Normal (d dimensions)
$$p(\mathbf{x}) = \frac{1}{(2\pi)^{d/2} |\boldsymbol{\Sigma}|^{1/2}} \exp\left(-\frac{1}{2}(\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})\right)$$

$\mathbf{x} \sim \mathcal{N}_d(\boldsymbol{\mu}, \boldsymbol{\Sigma})$. $\boldsymbol{\Sigma}$ must be symmetric positive definite (multivariate analog of $\text{Var}(X) > 0$). If singular, $|\boldsymbol{\Sigma}| = 0$ and the density is degenerate.

Bivariate Case ($d = 2$)

$$\boldsymbol{\mu} = \begin{pmatrix} \mu_1 \\ \mu_2 \end{pmatrix}, \quad \boldsymbol{\Sigma} = \begin{pmatrix} \sigma_1^2 & \rho\sigma_1\sigma_2 \\ \rho\sigma_1\sigma_2 & \sigma_2^2 \end{pmatrix}$$

The isoprobability contours are ellipses centered at $\boldsymbol{\mu}$:

  • Independent variables ($\rho = 0$): axes parallel to input axes; circle if $\sigma_1 = \sigma_2$, ellipse if different
  • Positive correlation ($\rho > 0$): major axis tilted positively
  • Negative correlation ($\rho < 0$): major axis tilted negatively
Isoprobability contours of bivariate normal: circle (equal, independent), axis-aligned ellipse (unequal, independent), tilted ellipse (correlated).

Special Case: Independent Components

If all components of $\mathbf{x}$ are independent, $\boldsymbol{\Sigma}$ is diagonal and the joint density is the product of individual univariate densities: $p(\mathbf{x}) = \prod_{j=1}^{d} p(x_j)$.

Linear Projection

If $\mathbf{W}$ is a $d \times k$ matrix with rank $k < d$ and $\mathbf{z}=\mathbf{W}^T\mathbf{x}$, then $\mathbf{z}$ follows a $k$-variate normal: $\mathbf{z} \sim \mathcal{N}_k(\mathbf{W}^T\boldsymbol{\mu}, \mathbf{W}^T\boldsymbol{\Sigma}\mathbf{W})$. Projecting a $d$-dimensional normal to $k$ dimensions preserves normality.

2.9 Multivariate Classification

With multivariate data, class-conditional densities are multivariate Gaussians: $p(\mathbf{x}|C_i) \sim \mathcal{N}_d(\boldsymbol{\mu}_i, \boldsymbol{\Sigma}_i)$.

Quadratic Discriminant Function
$$g_i(\mathbf{x}) = -\frac{1}{2}(\mathbf{x} - \boldsymbol{\mu}_i)^T\boldsymbol{\Sigma}_i^{-1}(\mathbf{x} - \boldsymbol{\mu}_i) - \frac{1}{2}\log|\boldsymbol{\Sigma}_i| + \log P(C_i)$$

Parameters to estimate: $K \cdot d$ for means, $K \cdot d(d+1)/2$ for covariance matrices.

Common Covariance ($\boldsymbol{\Sigma}_i = \boldsymbol{\Sigma}$ for all $i$) → Linear Discriminant

When all classes share the same covariance matrix, the quadratic terms cancel, and we get linear discriminant boundaries:

$$g_i(\mathbf{x}) = -\frac{1}{2}(\mathbf{x} - \boldsymbol{\mu}_i)^T\boldsymbol{\Sigma}^{-1}(\mathbf{x} - \boldsymbol{\mu}_i) + \log P(C_i)$$

If priors are equal, the optimal rule is to assign input to the class whose Mahalanobis distance is smallest.

Mahalanobis Distance
$$d_M(\mathbf{x}, \boldsymbol{\mu}_i) = \sqrt{(\mathbf{x} - \boldsymbol{\mu}_i)^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu}_i)}$$

Accounts for correlations and different scales of features, unlike Euclidean distance.

Naive Bayes Classifier

Assumes all off-diagonals of $\boldsymbol{\Sigma}$ are zero (independent features). Then $\boldsymbol{\Sigma}$ and $\boldsymbol{\Sigma}^{-1}$ are diagonal:

$$g_i(\mathbf{x}) = -\sum_{j=1}^{d} \frac{(x_j - \mu_{ij})^2}{2\sigma_{ij}^2} - \sum_{j=1}^{d}\frac{1}{2}\log\sigma_{ij}^2 + \log P(C_i)$$

Nearest Mean Classifier

If we further assume all variances are equal ($\sigma_i^2 = \sigma^2$ for all $i, j$), Mahalanobis distance reduces to Euclidean distance. With equal priors, classify $\mathbf{x}$ to the class with nearest mean. Only $Kd + 1$ parameters needed.

Simplification Hierarchy
Assumption Classifier Boundary
Different $\boldsymbol{\Sigma}_i$ per class Quadratic Discriminant Quadratic surfaces
Shared $\boldsymbol{\Sigma}$, equal priors Linear Discriminant (Mahalanobis) Hyperplanes
Diagonal $\boldsymbol{\Sigma}$ (independent features) Naive Bayes Axis-aligned
$\boldsymbol{\Sigma} = \sigma^2 \mathbf{I}$, equal priors Nearest Mean (Euclidean) Voronoi boundaries

2.10 Tuning Complexity: Regularized Discriminant Analysis (RDA)

There is a trade-off between the simplicity of a model and its generality. Making simplifying assumptions about $\boldsymbol{\Sigma}$ reduces parameters but introduces bias.

RDA — Interpolated Covariance
$$\boldsymbol{\Sigma}_i^{(\alpha,\beta)} = (1 - \alpha)\left[(1-\beta)\boldsymbol{\Sigma}_i + \beta\boldsymbol{\Sigma}\right] + \alpha\sigma^2\mathbf{I}$$
$\alpha$ $\beta$ Classifier
0 0 Quadratic Discriminant (separate covariances)
0 1 Linear Discriminant (shared covariance)
1 0 Nearest Mean Classifier ($\sigma^2\mathbf{I}$)

In between these extremes, we get a variety of classifiers where $\alpha, \beta$ are optimized by cross-validation.

2.11 Discrete Features

Some applications have discrete attributes (e.g., color $\in$ {red, green, blue}, pixel $\in$ {0, 1}).

Discrete Feature Classification

For binary features with independence (Naive Bayes):

$$g_i(\mathbf{x}) = \sum_{j=1}^{d} \left[x_j \log \frac{p_{ij}}{1 - p_{ij}} + \log(1 - p_{ij})\right] + \log P(C_i)$$

where $p_{ij} = P(x_j = 1 | C_i)$ is estimated from training data. This gives a linear discriminant.

Application: Document Categorization

In the bag-of-words representation, choose $d$ words a priori. $\hat{p}_{ij}$ estimates the probability that word $j$ occurs in document type $i$.

For multinomial features $x_j \in \{v_1, v_2, \ldots, v_n\}$:

$$p_{ijk} = P(x_j = v_k | C_i)$$

If all attributes are independent: $g_i(\mathbf{x}) = \sum_j \log p_{ijk(j)} + \log P(C_i)$ where $k(j)$ is the value of attribute $j$.

2.12 Multivariate Regression

The numeric output $r$ is a weighted sum of several input variables:

Multiple Linear Regression
$$r = w_0 + \sum_{j=1}^{d} w_j x_j + \epsilon = \mathbf{w}^T\mathbf{x} + \epsilon$$

where $\mathbf{x} = (1, x_1, x_2, \ldots, x_d)^T$ includes the bias term and $\epsilon \sim \mathcal{N}(0, \sigma^2)$.

Minimize MSE: $E = \sum_t (r^t - \mathbf{w}^T\mathbf{x}^t)^2$. Taking derivatives w.r.t. $w_j$ and setting to zero gives the normal equations:

Normal Equations (Matrix Form)
$$\mathbf{D}^T\mathbf{D}\mathbf{w} = \mathbf{D}^T\mathbf{r} \implies \mathbf{w} = (\mathbf{D}^T\mathbf{D})^{-1}\mathbf{D}^T\mathbf{r}$$
Knowledge Extraction from Linear Models
  • Look at signs of $w_j$: positive or negative effect on output
  • If all $x_j$ are in the same range, absolute values $|w_j|$ indicate feature importance
  • Features with $w_j \approx 0$ can be removed (feature selection)

2.13 Gradient Descent

In many cases, there is no analytical solution for the optimal parameters, and we resort to iterative optimization. The most common is gradient descent.

Definition

When $E(\mathbf{w})$ is a differentiable function of a vector of variables, the gradient vector is composed of partial derivatives:

$$\nabla E = \left(\frac{\partial E}{\partial w_1}, \frac{\partial E}{\partial w_2}, \ldots, \frac{\partial E}{\partial w_d}\right)^T$$
Gradient Descent Update Rule
$$\Delta w_i = -\eta \frac{\partial E}{\partial w_i}$$

$\eta$ = learning rate (step size). We start from random $\mathbf{w}$ and update in the opposite direction of the gradient. When gradient = 0, we've found a minimum.

Gradient descent follows the steepest downhill direction iteratively until reaching a minimum.
Key Considerations
  • Learning rate $\eta$: Too small → slow convergence. Too large → oscillation or divergence.
  • Finds the nearest minimum — may be a local minimum, no guarantee of global minimum (unless the function is convex)
  • Gradient ascent maximizes a function (goes in direction of gradient)
  • More advanced: second-order methods, conjugate gradient (faster, more memory). Costly methods: simulated annealing, genetic algorithms (more thorough search).

2.14 Logistic Discrimination

Two Classes

Instead of estimating class-conditional densities separately, we directly model the posterior probability using the log likelihood ratio:

Assume the log likelihood ratio is linear:

Logistic Model
$$\log \frac{P(C_1|\mathbf{x})}{P(C_2|\mathbf{x})} = \mathbf{w}^T \mathbf{x} + w_0$$

Rearranging using Bayes' rule, we get the sigmoid (logistic) function:

Sigmoid Function
$$P(C_1|\mathbf{x}) = \sigma(\mathbf{w}^T\mathbf{x} + w_0) = \frac{1}{1 + \exp(-(\mathbf{w}^T\mathbf{x} + w_0))}$$
The sigmoid function σ(z) maps any real number to (0, 1). Decision: choose C₁ if σ(z) > 0.5 (i.e., z > 0).

Training

Given a sample of two classes, the sample likelihood is:

$$E = -\log \ell = -\sum_{t=1}^{N} \left[r^t \log y^t + (1 - r^t)\log(1 - y^t)\right]$$

This is the cross-entropy loss. We minimize $E$ using gradient descent. There is no closed-form solution.

Practical Tips
  • Z-normalization before training: for each input $j$, compute mean $m_j$ and std $s_j$, normalize: $\tilde{x}_j = (x_j - m_j)/s_j$. Same $m_j, s_j$ from training set used for test data.
  • After training, during testing: choose $C_1$ if $y > 0.5$, else $C_2$
  • Early stopping (stopping before zero training error) is a form of regularization — weights stay close to zero.

Multiple Classes ($K > 2$): Softmax

For $K$ classes, we generalize using the softmax function:

Softmax Function
$$y_i = P(C_i|\mathbf{x}) = \frac{\exp(\mathbf{w}_i^T\mathbf{x})}{\sum_{k=1}^{K}\exp(\mathbf{w}_k^T\mathbf{x})}$$
Softmax Properties
  • If the weighted sum for one class is sufficiently larger → after exponentiation and normalization, its $y_i \approx 1$ and others $\approx 0$
  • Works like a differentiable maximum (hence "softmax")
  • Guarantees $\sum_i y_i = 1$ and $0 \leq y_i \leq 1$ (valid probability distribution)
Numerical — Softmax Example

Given: 3 classes with scores $z_1 = 2.0$, $z_2 = 1.0$, $z_3 = 0.1$

$e^{z_1} = e^{2.0} = 7.389$, $e^{z_2} = e^{1.0} = 2.718$, $e^{z_3} = e^{0.1} = 1.105$

$\text{Sum} = 7.389 + 2.718 + 1.105 = 11.212$

$$y_1 = \frac{7.389}{11.212} = 0.659, \quad y_2 = \frac{2.718}{11.212} = 0.242, \quad y_3 = \frac{1.105}{11.212} = 0.099$$

Decision: Classify as $C_1$ (highest probability 65.9%). Sum: $0.659 + 0.242 + 0.099 = 1.0$ ✓
