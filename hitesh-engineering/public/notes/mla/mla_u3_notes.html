8.1

Introduction

In previous chapters, we used parametric methods where we assumed a specific functional form for the underlying distribution (like a Gaussian). However, in many real-world scenarios, the actual distribution is unknown and may not fit standard models.

✦ Core Philosophy
Nonparametric methods do not assume a fixed functional form. Instead, they assume that similar inputs should have similar outputs.

These methods are often called memory-based learning or instance-based learning because they typically store the entire training dataset in memory. Unlike parametric models, there is no "training phase" in the traditional sense; instead, most of the computation happens at the time of prediction (query time).

Key Characteristics

  • Local Models: The estimate for a given input is influenced only by the training points in its immediate neighborhood.
  • Flexible: Can model complex, multi-modal distributions that parametric models might miss.
  • Data Requirement: As the dataset size increases, the model complexity grows to better fit the data.
8.2

Nonparametric Density Estimation

The objective of density estimation is to find an estimator \(\hat{p}(x)\) for an unknown probability density \(p(x)\) based on a sample \(\mathcal{X} = \{x^1, x^2, \dots, x^N\}\).

Interactive Density Estimation
Data Points Estimated Density \(\hat{p}(x)\)
The Smoothing Parameter (h)
The parameter \(h\) (window width or bin width) controls the bias-variance trade-off:
  • Small \(h\): High variance. The estimate is "spiky" and overfits the noise in the data.
  • Large \(h\): High bias. The estimate is too smooth and may miss critical features of the distribution.

8.2.1 Histogram Estimator

The simplest approach. The input space is divided into equal-sized "bins" of width \(h\). Given an origin \(x_0\), the bins are defined by intervals \([x_0 + mh, x_0 + (m+1)h]\).

\[ \hat{p}(x) = \frac{\#\{x^t \text{ in the same bin as } x\}}{Nh} \] → Where N is the total number of samples and h is the bin width.
Limitations
1. Discontinuity: The estimate jumps at bin boundaries.
2. Origin Dependency: The shape depends on the choice of \(x_0\).

Naive Estimator

To fix the origin dependency, we can center the window of width \(h\) at the point \(x\). This is the naive estimator:

\[ \hat{p}(x) = \frac{1}{Nh} \sum_{t=1}^N w\left(\frac{x - x^t}{h}\right) \] where \(w(u)\) is a weight function (rectangular window): \[ w(u) = \begin{cases} 1 & \text{if } |u| < 1/2 \\ 0 & \text{otherwise} \end{cases} \]

8.2.2 Kernel Estimator (Parzen Window)

The Naive estimator is still discontinuous. To get a smooth estimate, we replace the box weight function with a smooth Kernel Function (K).

\[ \hat{p}(x) = \frac{1}{Nh} \sum_{t=1}^N K\left(\frac{x - x^t}{h}\right) \]

The most common choice is the Gaussian Kernel:

\[ K(u) = \frac{1}{\sqrt{2\pi}} \exp\left(-\frac{u^2}{2}\right) \]

Using a Gaussian kernel results in a density estimate that is the sum of \(N\) Gaussians, each centered at a training point \(x^t\), with variance \(h^2\).

8.2.3 k-Nearest Neighbor (k-NN) Estimator

In previous methods, we fixed the volume (width \(h\)) and counted how many samples fell inside. In the k-NN estimator, we fix the number of samples \(k\) and let the volume vary.

\[ \hat{p}(x) = \frac{k}{2N d_k(x)} \] → Where \(d_k(x)\) is the distance from \(x\) to its \(k\)-th nearest neighbor.

This method adapts to the density of the data: it uses a small window in high-density regions and a large window in low-density regions.

8.3

Generalization to Multivariate Data

For \(d\)-dimensional data (vectors \(\mathbf{x}\)), the density estimators are extended by scaling the volume appropriately.

Multivariate Kernel Estimator

\[ \hat{p}(\mathbf{x}) = \frac{1}{Nh^d} \sum_{t=1}^N K\left(\frac{\mathbf{x} - \mathbf{x}^t}{h}\right) \]

A common kernel is the Multivariate Gaussian. If the variables are independent, the kernel can simply be a product of univariate kernels.

Multivariate k-NN Estimator

\[ \hat{p}(\mathbf{x}) = \frac{k}{N V_k(\mathbf{x})} \] where \(V_k(\mathbf{x})\) is the volume of a hypersphere with radius \(r = d_k(\mathbf{x})\).

The volume is calculated as \(V_k(\mathbf{x}) = c_d \cdot (d_k(\mathbf{x}))^d\), where \(c_d\) is a constant for the volume of a unit sphere in \(d\) dimensions.

8.4

Nonparametric Classification

Nonparametric classification applies the density estimation techniques from Section 8.2 to the problem of predicting class labels. We use Bayes' Rule to calculate the posterior probability for each class \(C_i\).

\[ g_i(x) = \hat{p}(x|C_i) \hat{P}(C_i) \] → Where \(\hat{p}(x|C_i)\) is the class-conditional density and \(\hat{P}(C_i) = N_i/N\) is the prior.

k-Nearest Neighbor (k-NN) Classifier

The k-NN classifier is the most popular nonparametric classification method. Instead of estimating densities explicitly, it simplifies the posterior probability calculation:

\[ \hat{P}(C_i|x) = \frac{k_i}{k} \] → Where \(k_i\) is the number of neighbors from class \(C_i\) among the \(k\) nearest points.
The Decision Rule
Assign the input \(x\) to the class that is most heavily represented among its \(k\) nearest neighbors (Majority Vote).
8.5

Condensed Nearest Neighbor

The primary disadvantage of k-NN is its space and time complexity: it requires storing all \(N\) training points and calculating distances to all of them for every prediction. Condensed Nearest Neighbor (CNN) is a data reduction algorithm that finds a small subset \(\mathcal{Z}\) of the training set \(\mathcal{X}\) that performs as well as the full set.

The Core Logic
Points that are deep inside a class region (far from the decision boundary) do not contribute to the classification boundary and can be safely removed. We only need to store points near the boundary.

The CNN Algorithm

  1. Initialize \(\mathcal{Z}\) as an empty set (or add one random instance from each class).
  2. For each instance \(x\) in the training set \(\mathcal{X}\):
    • Find its nearest neighbor \(x'\) in the subset \(\mathcal{Z}\).
    • If \(class(x) \neq class(x')\) (i.e., \(\mathcal{Z}\) misclassifies \(x\)), add \(x\) to \(\mathcal{Z}\).
  3. Repeat the loop until no more points are added to \(\mathcal{Z}\) in a full pass.
8.6

Distance-Based Classification

Distance-based classifiers assign instances to a class based on their proximity to class representatives (like means) or specific training members.

Common Distance Measures

  • Euclidean Distance: The most common measure for continuous features. \[ \mathcal{D}(x, y) = \sqrt{\sum_{j=1}^d (x_j - y_j)^2} \]
  • Mahalanobis Distance: Accounts for the covariance \(S\) and scaling of different features. \[ \mathcal{D}(x, m_i) = (x - m_i)^T S^{-1} (x - m_i) \]
  • Manhattan Distance (L1): The sum of absolute differences. \[ \mathcal{D}_1(x, y) = \sum_{j=1}^d |x_j - y_j| \]
  • Hamming Distance: Used for discrete or binary features; counts the number of mismatching attributes.
Nearest Mean Classifier
A simple distance-based classifier that assigns \(x\) to the class \(C_i\) whose mean \(m_i\) is closest to \(x\).
8.7

Outlier Detection

Outliers are data points that don't follow the general pattern of the rest of the data. They might be caused by measurement errors (noise) or represent rare, interesting events (anomalies).

k-NN Based Detection

  • Distance-based: A point \(x\) is an outlier if its distance to its \(k\)-th nearest neighbor is greater than a threshold \(\theta\). \[ d_k(x) > \theta \implies \text{Outlier} \]
  • Density-based: A point is an outlier if it has fewer than \(k_{min}\) neighbors within a fixed radius \(R\).
Applications
Outlier detection is critical for Fraud Detection (identifying unusual transactions) and Data Cleaning (removing noise before training).
8.8

Nonparametric Regression: Smoothing Models

In nonparametric regression, we assume that the output function \(g(x)\) is smooth, meaning points close to \(x\) should have similar response values. We estimate \(g(x)\) by taking a weighted average of the training responses \(r^t\) in the neighborhood of \(x\).

8.8.1 Running Mean Smoother

A simple average of response values within a symmetric window (bin) of width \(h\) around \(x\).

\[ \hat{g}(x) = \frac{\sum_{t=1}^N w\left(\frac{x-x^t}{h}\right) r^t}{\sum_{t=1}^N w\left(\frac{x-x^t}{h}\right)} \] where \(w(u) = 1\) if \(|u| < 1\), and \(0\) otherwise.

8.8.2 Kernel Smoother

Also known as the Nadaraya-Watson estimator. It uses a smooth kernel \(K(\cdot)\) to give higher weight to points closer to \(x\).

\[ \hat{g}(x) = \frac{\sum_{t=1}^N K\left(\frac{x-x^t}{h}\right) r^t}{\sum_{t=1}^N K\left(\frac{x-x^t}{h}\right)} \]

For a **Gaussian kernel**, this estimate is a surface that "smooths" out the data points into a continuous curve.

8.8.3 Running Line Smoother

The running mean often suffers from bias at the boundaries of the data. The running line smoother (or locally weighted regression) fixes this by fitting a local linear model instead of just a constant mean.

\[ \text{Minimize: } E(a, b|x) = \sum_{t=1}^N w\left(\frac{x-x^t}{h}\right) [r^t - (a + b(x^t - x))]^2 \] The estimate at \(x\) is simply the intercept \(\hat{g}(x) = a\).
9.1

Introduction to Decision Trees

A Decision Tree is a hierarchical model for supervised learning. It consists of internal nodes (tests/splits) and leaf nodes (outputs). It works by recursively partitioning the input space into local regions.

  • Root Node: The first split point (contains all data).
  • Internal Nodes: Points where a specific test is applied to a feature.
  • Leaves: Final nodes that store the class label (Classification) or mean value (Regression).
9.2

Univariate Trees

In a univariate tree, each internal node uses only one input dimension \(x_j\). For numeric data, this is typically a binary split: \(x_j > w_{m0}\).

9.2.1 Classification Trees (Impurity)

The goal is to split a node such that the resulting child nodes are as "pure" as possible (containing instances from mostly one class). Impurity \(I(m)\) measures how mixed the classes are at node \(m\).

Impurity Measures
Let \(p_m^i\) be the fraction of instances at node \(m\) that belong to class \(C_i\).
  1. Entropy: \[ I(m) = -\sum_{i=1}^K p_m^i \log_2 p_m^i \]
  2. Gini Index: \[ I(m) = 1 - \sum_{i=1}^K (p_m^i)^2 \]
  3. Misclassification Error: \[ I(m) = 1 - \max_i p_m^i \]

We choose the split that maximizes the Information Gain (the reduction in total impurity across child nodes).

9.2.2 Regression Trees

In regression, "impurity" is measured by the **Variance** of the response values at the node.

\[ I(m) = \sigma_m^2 = \frac{1}{N_m} \sum_{t \in \mathcal{X}_m} (r^t - \bar{r}_m)^2 \] → Where \(\bar{r}_m\) is the average response value at node \(m\).

The output of a leaf in a regression tree is simply the mean value \(\bar{r}_m\) of all training points that reach that leaf.

9.3

Pruning

Trees are prone to overfitting. A tree that is too deep will capture noise in the training data. Pruning simplifies the tree to improve generalization.

  • Pre-pruning (Early Stopping): Stop growing the tree if the number of instances in a node falls below a threshold, or if the impurity reduction is not statistically significant.
  • Post-pruning: Grow the full tree (possibly to 0 impurity) then remove subtrees starting from the bottom. We replace a subtree with a leaf if the error on a **validation set** does not increase.
Error-Complexity Pruning
A typical strategy where we minimize a cost function that combines the error rate and the number of nodes: \(Cost = Error + \alpha \cdot (\text{Size})\).
9.4 & 9.5

Rule Extraction and Learning Rules

Rule Extraction (9.4)

Every path from the root to a leaf in a decision tree can be written as an IF-THEN rule. The set of all such rules is exhaustive (covers all cases) and mutually exclusive (only one rule applies to any instance).

Learning Rules from Data (9.5)

Instead of building a tree, we can learn rules directly using a Sequential Covering (separate-and-conquer) algorithm:

  1. Find the best single rule that covers some positive instances.
  2. Remove the instances covered by this rule from the dataset.
  3. Repeat until all instances are covered.

14.1

Introduction to Kernel Machines

Kernel Machines, such as Support Vector Machines (SVMs), are among the most powerful algorithms in machine learning. They bridge the gap between global models (like linear regression) and local models (like k-NN). While they find a global decision boundary, that boundary is defined only by a few "local" critical points called Support Vectors.

14.2

Optimal Separating Hyperplane

For a linearly separable dataset, there are infinitely many hyperplanes that can separate the classes. SVM chooses the one that has the **Maximum Margin**.

\[ \text{Minimize: } \frac{1}{2} \|w\|^2 \] \[ \text{Subject to: } r^t(w^T x^t + w_0) \ge 1 \] The margin width is exactly \(\frac{2}{\|w\|}\).

Support Vectors
The training points \(x^t\) that satisfy the constraint with equality (\(r^t(w^T x^t + w_0) = 1\)) are the Support Vectors. They are the only points that "support" the boundary; removing other points wouldn't change the boundary.
14.3

The Nonseparable Case: Soft Margin

If the data is not linearly separable, we introduce Slack Variables (\(\xi^t\)) to allow some points to be on the wrong side of the margin or even the boundary.

\[ \text{Minimize: } \frac{1}{2} \|w\|^2 + C \sum_{t=1}^N \xi^t \] → \(C\) is the penalty parameter. Large \(C\) = strict margin (overfitting risk), Small \(C\) = soft margin (underfitting risk).
14.5

The Kernel Trick

The Kernel Trick allows SVM to find a linear boundary in a high-dimensional feature space without ever explicitly mapping the data to that space. It substitutes the dot product \(x^T x'\) with a Kernel Function \(K(x, x')\).

\[ g(x) = \sum_{t=1}^N \alpha^t r^t K(x^t, x) + w_0 \]

14.7 Defining Kernels

  • Polynomial: \( K(x, x') = (x^T x' + 1)^q \)
  • Radial Basis Function (RBF): \( K(x, x') = \exp\left(-\frac{\|x - x'\|^2}{2\sigma^2}\right) \)
  • Sigmoid: \( K(x, x') = \tanh(\eta x^T x' + \nu) \)
14.9 - 14.12

Extensions of Kernel Machines

Multiclass Kernel Machines (14.9)

  • One-vs-All: Train \(K\) classifiers, one for each class vs the rest.
  • One-vs-One: Train \(\frac{K(K-1)}{2}\) pairwise classifiers.

Kernel Machines for Regression (14.10)

Uses the \(\epsilon\)-sensitive loss function, where errors smaller than \(\epsilon\) are ignored. The goal is to find a function that stays within a "tube" of width \(2\epsilon\) around the data.

One-Class Kernel Machines (14.12)

Used for Novelty/Outlier Detection. It finds a boundary that encloses the high-density region of the data. Anything outside this boundary is flagged as an anomaly.

14.13 & 14.14

Dimensionality Reduction

Large Margin Nearest Neighbor (14.13)

Learns a distance metric to improve k-NN classification by pulling "target neighbors" closer and pushing "imposters" (different class neighbors) away.

Kernel PCA (14.14)

Performs Principal Component Analysis in the high-dimensional feature space defined by a kernel. This allows for **Nonlinear Dimensionality Reduction**.
